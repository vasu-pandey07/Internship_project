EduPlatform Project Study Notes (MERN)

Project Type: Online learning platform (Coursera/Udemy style) with student, instructor, and admin roles.

React + Vite Redux Toolkit Node.js + Express MongoDB + Mongoose JWT Auth Stripe Socket.io

1. Project Overview

What this project does

  • Students can register/login, browse courses, enroll (free/paid), watch lessons, take quizzes, submit assignments, ask questions, and get certificates.
  • Instructors can create/edit courses, add lessons/quizzes/assignments, answer student questions, and submit courses for admin approval.
  • Admin can review/approve/reject courses, manage users/roles, view analytics, and moderate content.

Frontend

  • React Router for page routing
  • Redux Toolkit slices for auth/courses/enrollments
  • Axios instance with JWT interceptor
  • Role-based protected routes

Backend

  • Express API with controllers/routes/middleware
  • Mongoose models for users, courses, enrollments, quizzes, etc.
  • JWT auth + RBAC middleware
  • Stripe for paid courses, Socket.io for notifications

High-level architecture

React UI (client)
  -> Axios API calls (/api/v1/...)
    -> Express Routes -> Controllers -> Services
      -> Mongoose Models -> MongoDB
    <- JSON response
  <- Redux state update + UI render

4. Important Code Explanations

Function: `auth` middleware (JWT verification)

const auth = async (req, res, next) => {
  const authHeader = req.headers.authorization;
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({ success: false, message: 'Access denied. No token provided.' });
  }

  const token = authHeader.split(' ')[1];
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  const user = await User.findById(decoded.id);
  if (!user) return res.status(401).json({ success: false, message: 'Token is valid but user no longer exists.' });

  req.user = user;
  next();
};

Explanation:

  • Line 1-4: Checks if token exists in `Authorization` header.
  • Line 6: Extracts raw JWT token.
  • Line 7: Verifies token signature and decodes payload (`id`, `role`).
  • Line 8-9: Ensures user still exists in DB.
  • Line 11-12: Attaches user to request and allows next middleware/controller.

Why used: Protects private APIs and enables role checks later (`rbac`).

Function: `updateProgress()` (learning progress + certificate unlock)

const lessonProgress = enrollment.progress.find(
  (p) => p.lesson.toString() === lessonId
);
lessonProgress.completed = true;
lessonProgress.watchedAt = new Date();

const totalLessons = enrollment.progress.length;
const completedLessons = enrollment.progress.filter((p) => p.completed).length;
enrollment.completionPercentage = Math.round((completedLessons / totalLessons) * 100);

const eligibility = await evaluateEnrollmentCertificateEligibility({
  courseId: enrollment.course,
  studentId: enrollment.student,
  progress: enrollment.progress,
});

if (eligibility.eligible && !enrollment.completedAt) {
  enrollment.completedAt = new Date();
}

Explanation:

  • Finds matching lesson in enrollment progress array.
  • Marks lesson completed and sets timestamp.
  • Recalculates percent completion.
  • Calls certificate service to check both rules: all lessons complete + all quizzes passed.
  • If eligible, sets `completedAt` (used by certificate API).

Why used: Central place to keep progress, completion, and certificate logic consistent.

Function: `createCheckout()` (Stripe payment start)

const session = await stripe.checkout.sessions.create({
  payment_method_types: ['card'],
  line_items: [{
    price_data: {
      currency: 'usd',
      product_data: { name: course.title },
      unit_amount: Math.round(course.price * 100),
    },
    quantity: 1,
  }],
  mode: 'payment',
  success_url: `${clientUrl}/payment/success?session_id={CHECKOUT_SESSION_ID}`,
  cancel_url: `${clientUrl}/payment/cancel`,
  metadata: {
    courseId: course._id.toString(),
    studentId: req.user._id.toString(),
  },
});

Explanation:

  • Creates one-time Stripe checkout session for selected course.
  • Converts dollar amount to cents (`*100`) because Stripe expects smallest currency unit.
  • Sets success/cancel URLs to frontend routes.
  • Stores `courseId` and `studentId` in metadata for later verification and auto-enrollment.

Why used: Securely handles paid enrollment without storing card details on your server.

Function: Axios request interceptor (frontend auth)

API.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

Explanation:

  • Runs before every API request.
  • Reads JWT from localStorage.
  • Adds `Authorization: Bearer <token>` header automatically.

Why used: Avoids writing token header manually in every component.

5. Data Flow (Frontend -> API -> Backend -> DB -> Response)

Example: Student enrolls in a free course

  1. Student clicks Enroll Now in `CourseDetail.jsx`.
  2. Frontend dispatches `enrollInCourse(courseId)` thunk.
  3. Axios sends `POST /api/v1/enrollments` with JWT header.
  4. `auth` middleware validates JWT, sets `req.user`.
  5. `enrollmentController.enroll` validates course, checks existing enrollment, checks price.
  6. Backend creates enrollment document + initial lesson progress array.
  7. Course `totalStudents` increments and notification created.
  8. JSON response returns new enrollment.
  9. Redux slice updates store; UI reflects enrolled state.

Example: Paid course flow

  1. Frontend calls `POST /payments/create-checkout`.
  2. Backend creates Stripe checkout session and pending payment record.
  3. User pays on Stripe page.
  4. Frontend opens `/payment/success?session_id=...` page.
  5. `PaymentSuccess.jsx` calls `GET /payments/verify/:sessionId`.
  6. Backend confirms Stripe session status and marks payment `completed`.
  7. Backend auto-creates enrollment and updates course stats.
  8. Student sees success UI and can start course.

6. Key Concepts Used

JWT Authentication

  • JWT created in `User.generateToken()`.
  • Sent after register/login.
  • Stored in localStorage and attached by Axios interceptor.
  • Verified by backend `auth` middleware.

Redux Toolkit

  • Slices: `auth`, `courses`, `enrollments`.
  • `createAsyncThunk` handles async API calls.
  • Store updates trigger reactive UI render.

Middleware

  • `auth` and `rbac` for secure role access.
  • `errorHandler` for standard API error responses.
  • `upload` for image/video/document files.
  • Rate limiting, CORS, Helmet for API security.

Database Schema Design

  • Relations with ObjectId refs (User -> Course, Course -> Lesson, etc.).
  • Indexes for speed (email, status, category, compound unique indexes).
  • Domain-driven collections: enrollments, quiz attempts, notifications.

7. Important APIs

Endpoint Method Purpose Request Response (main data)
/api/v1/auth/register POST Create account { name, email, password, role } { user, token }
/api/v1/auth/login POST Login user { email, password } { user, token }
/api/v1/courses GET List approved courses with filters query params (search, category, page...) { courses, pagination }
/api/v1/courses/:id GET Get one course + lessons path param id { course }
/api/v1/enrollments POST Enroll in free course { courseId } { enrollment }
/api/v1/enrollments/:id/progress PUT Mark lesson complete { lessonId } { enrollment }
/api/v1/payments/create-checkout POST Create Stripe session { courseId } { sessionId, url }
/api/v1/payments/verify/:sessionId GET Verify payment and enroll path param sessionId { status }
/api/v1/courses/:courseId/quizzes/:id/attempt POST Submit quiz attempt { answers: number[] } { attempt, correctAnswers }
/api/v1/admin/courses/:id/status PUT Approve/reject course { status, reason? } { course }
/api/v1/notifications GET List user notifications query params { notifications, unreadCount, pagination }
